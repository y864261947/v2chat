# 伴侣功能详细规格（Feature Spec）

- 版本：v0.1（把 2026-07-04 需求访谈中敲定的全部细节固化为 UX 级规格）
- 标记约定：**[已定]** 访谈明确决策 · **[默认]** 我方建议默认值，可推翻 · **[待定]** 尚未讨论，实现前需拍板
- 上位文档：[android-companion-2dev.md](./android-companion-2dev.md)；协议见 [backend-api-spec.md](./backend-api-spec.md)

## 1. 角色模型（一切功能的载体）

**[已定]** 多角色微信式：App 像微信，每个 AI 角色像一个联系人。角色基于现有 Copilots 系统扩展。

角色数据结构（在现有 copilot 字段上新增）：

| 字段 | 类型 | 说明 |
|---|---|---|
| `voiceId` | string? | TTS 音色（内置音色名或克隆 voice_id），空=不发语音 |
| `stickerPackId` | string? | 角色专属表情包库 id，空=不发表情 |
| `memoryEnabled` | boolean | 该角色是否开启记忆（默认 true）**[默认]** |
| `greeting` | string? | 开场白（新会话第一条 AI 消息）**[默认]** |

角色创建/编辑页新增区块：声音（选内置/克隆/无）、表情包（进入分配页）、记忆开关。

## 2. 首页信息架构

**[已定]** 首页 = 会话列表 = "联系人列表"。每项：角色头像（大）、角色名、最后一条消息预览、时间。

- 消息预览规则 **[默认]**：语音消息显示 `[语音]`，表情包显示 `[表情]`，其余取文本前 30 字
- **[已定]** 一个角色一个主会话（微信隐喻）；Chatbox 原有多线程/分叉能力保留但入口弱化 **[默认]**
- **[待定→前端开发者决定]** 底部 Tab 结构（如 会话/发现角色/我的）还是保留侧栏抽屉——由前端在 M2.1 动工前拍板并回写到这里（D19）
- **[默认]** 不做未读角标（本地应用无推送，语义不成立；主动消息上线后再议）
- **[已定]**（M1 执行）：官方 Getting Started 引导停用；M2.1 换成「创建你的第一个角色」引导

## 3. 语音消息（微信式，双向）

### 3.1 用户发语音 [已定]

- 输入栏麦克风按钮，**按住说话**，上滑取消，松开发送 **[默认手势，对齐微信]**
- 录音上限 60s **[默认]**；到限自动截断发送
- 技术：WebView `MediaRecorder`（需原生侧授予 RECORD_AUDIO + WebView 权限桥接）
- 发送链路：录音落 blob 存储 → 调 STT（协议 §4.2）得转写 → 消息以 **audio part** 入库：`{ type:'audio', blobKey, durationMs, transcript }`
- 模型侧：`context/builder` 把 audio part 以 transcript 文本喂给模型（模型无感语音）
- STT 失败降级 **[默认]**：气泡显示"转写失败·点按重试"，消息可发但不入上下文，重试成功后补

### 3.2 AI 回语音 [已定]

- 角色配置了 `voiceId` 且用户开启"语音回复"时：AI 文本回复完成后调 TTS（协议 §4.3，voice=角色音色）→ 存 blob → 渲染为语音气泡
- 每角色开关 + 全局开关（设置页）**[默认]**
- 流式期间先出文字，TTS 完成后气泡切换为语音形态（文字折叠可展开）**[默认]**

### 3.3 语音气泡交互 [默认]

- 样式：播放键 + 波形/时长（如 `0:23`）；播放中高亮进度
- 点按播放/暂停；长按菜单：查看文字、重新转写、删除
- 自动播放：进入会话不自动播；停留会话中新收到的 AI 语音自动播（可在设置关）
- 同屏只有一个音频在播（新播停旧）

## 4. 音色克隆 [已定]

- 入口：角色编辑 → 声音 → 「克隆新音色」
- 流程：录制或上传样本（10–60s，≤10MB）→ **勾选授权声明**（"我确认有权使用该声音"，协议 consent 字段，必勾）→ 上传 → `processing` 状态轮询（UI 显示"克隆中…"）→ `ready` 后出现在音色列表可选
- 音色归属用户账号，可跨角色复用、可删除；配额 10 个（后端）
- 内置音色：后端下发清单（中文伴侣向，供应商倾向 fish-audio）**[已定方向]**
- 合规红线：克隆知名声优/角色音色的版权灰区——公开分发前评估（风险清单 #5）

## 5. 表情包（双向）[已定]

### 5.1 库模型

- **用户库**：用户自己发表情时从这里选
- **角色库**：每个角色一套「TA 的表情包」，由用户帮角色添加；AI 只能从自己那套里挑（人设一致性）
- 预置：内置少量（≤20 张）无版权风险表情，避免开箱体验空白

### 5.2 导入流程

相册选图（可多选）→ 压缩转码（**[默认]** webp、最长边 512px、≤200KB）→ 视觉模型自动生成一句话描述（可编辑）→ 选择入用户库/某角色库

描述是给 AI 选表情用的，质量决定 AI 发表情的贴切度。

### 5.3 发送

- 用户发：输入栏表情按钮 → picker 网格（最近使用置顶 **[默认]**）→ 以 image part 发送 → 视觉模型可"看懂"**[已定]**
- AI 发：注册 `send_sticker` 工具（参数 `sticker_id`，enum 限定为该角色库；工具描述附每张贴纸的 description）；模型自主决定何时调用；渲染为表情消息（大图无气泡框 **[默认]**）
- 无视觉能力模型收到用户表情时 **[默认]**：以 `[表情：<description>]` 文本替代喂入

## 6. 记忆系统

### 6.1 基础：本地记忆 + 记忆册 [已定]

- **提取**：后台用当前会话模型从对话中提取长期事实（名字、喜好、近况、纪念日、关系进展）
  - 触发 **[默认]**：会话切后台 / 空闲 2 分钟 / 距上次提取新增 ≥20 条消息，满足其一
  - 提取时把该角色已有记忆一并给模型，让其输出 新增/更新/删除 三类操作（避免重复堆积）
- **存储**：sqlite 表，字段 **[默认]**：`id, roleId, content, category(fact|preference|event|relationship), sourceSessionId, createdAt, updatedAt`
- **注入**：组装 system prompt 时带入该角色记忆（**[默认]** 上限 50 条或 2000 字符，超限按 updatedAt 新者优先）
- **记忆册页面**：按角色分组；查看/编辑/删除/手动添加；条目显示分类与时间 **[已定]**
- **[已定]** 记忆按角色隔离（A 角色不知道 B 角色的事）
- **[待定]** 是否引入跨角色共享的"用户档案"层（名字生日等硬事实全角色可见）——M2.5 动工前定
- 删除会话不删记忆；删除角色时询问是否连带删除记忆 **[默认]**

### 6.2 高级：Honcho 云记忆 [已定]

- Honcho **托管版**，客户端经自研后端代理访问（协议 §8 占坑，M4 详设），客户端不持有 Honcho key
- **opt-in**：默认关闭；开启入口在角色记忆设置，需展示隐私提示（对话内容将上传至云端记忆服务）并二次确认
- 可作为付费/会员能力（绑积分）
- 本地/云端同为 `MemoryController` 接口的两个实现，切换不动聊天管线

## 7. 账号 / 积分 / 卡密 [已定]

- **未登录可用**：自接中转站/OpenAI 兼容网关（Chatbox 原生自定义 provider）不要求登录；只有默认自研服务需要
- 登录：邮箱 → 收 6 位验证码 → 登录（首登即注册）；协议 §2
- 「我的」页：邮箱、积分余额、卡密兑换输入框、检查更新、意见反馈、退出登录 **[默认]**
- 余额展示：聊天页不常驻显示 **[默认]**，`402 insufficient_credits` 时弹充值引导（跳卡密兑换）
- 计费认知：积分按 token/语音秒/合成字符扣（协议 §3/§4），单价由后端下发，客户端不硬编码

## 8. 主动消息（后置，形态先记录）[已定后置]

- 一期不做。远期形态：本地定时通知（早晚问候、N 小时未互动），点开后 AI 基于记忆即时生成开场白
- 更远期愿景：接入穿戴/健康等个人数据源驱动主动关怀（协议已占坑 `/api/datasources/*`）

## 9. 非功能需求

- **零遥测** [已定]：无崩溃上报、无行为统计；唯一回传=用户主动反馈（协议 §6，可附本地日志）
- 隐私立场：本地优先（对话/记忆/表情包全在设备），云端仅推理与可选 Honcho——这是产品卖点，文案要体现
- 离线行为 **[默认]**：无网可浏览全部历史与记忆册；发送失败气泡红色感叹号可重试
- 冷启动：强杀后启动白屏需优化（splash 隐藏时机，M2.1 顺手）
- 性能基线：中端机（4GB RAM）流畅，语音气泡列表滚动不掉帧

## 10. Parked（明确不做，防 scope 蔓延）
